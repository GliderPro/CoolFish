CoolFishNS.FiniteStateMachine.States NamespaceCoolFish Plugin API
The CoolFishNS.FiniteStateMachine.States namespace contains classes for mangaging all of the default states built into the state. These states can be used by default or you can use your own
Classes
  ClassDescription
State
Abstract State class that all states should inherit from and implement the required members
StateApplyLure
State which handles applying a fishing lure if we need one
StateBobbing
This state handles if the fishing bobber actively has a fish on the line.
StateDoLoot
This state is run if we caught a fish and have items in the loot window
StateDoNothing
This state is run if we are moving or get into combat. This prevents the bot from trying to do anything when it is unable to do so.
StateDoWhisper
This state is run if we want to be notified by whispers in the game and one occurs.
StateFish
This state is run if we have nothing else to do and we aren't casting the "fishing" spell already
StateStopOrLogout
This state contains all of the conditions in which we might stop the bot and/or logout
StateUseCharm
State which handles applying the Ancient Pandaren Fishing Charm if we need one
StateUseRaft
Run this state if we want to use water walking or Angler's raft item to fish in open water
StateUseRumsey
State which handles applying the Rumsey if we need it and have it
StateUseSpear
State which handles applying the Spear if we need it and have it
